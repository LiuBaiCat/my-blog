## 02.组合式API - setup选项

![](/img_05/02.png)

#### ①复杂写法(原理)

```vue
<!-- 加上 setup 允许在 script 中直接试用编写组合式API -->
<script>
  // setup
  // 1. 执行时机，比beforeCreate还要早
  // 2. setup函数中无法使用this（this 是 undefined）
  // 3. 数据和函数需要return出去，才能在模板中使用
export default {
  setup () {
    // console.log('setup函数')
    const message = 'hello Vue3'
    const logMessage = () => {
      console.log(message)
    }
    return {
      message,
      logMessage
    }
  },
  beforeCreate () {
    console.log('beforeCreate')
  }
}
</script>

<template>
  <div>{{ message }}</div>
  <button @click="logMessage">按钮</button>
</template>

```

#### ②简写 <script setup>语法糖

```vue
<script setup>
const message = 'this is a message'
const logMessage = () => {
  console.log(message)
}
</script>

<template>
  <div>{{ message }}</div>
  <button @click="logMessage">按钮</button>
</template>

```

## 03.组合式API - reactive和ref函数

![](/img_05/04.png)

![](/img_05/05.png)

```vue
<script setup>
  // reactive: 接受一个对象类型的数据，返回一个响应式的对象
  // 如果是简单写法呢？
  // import { reactive } from 'vue'

  // const state = reactive({
  //   count: 100
  // })
  // const setCount = () => {
  //   state.count++
  // }
  // 2.ref: 接受一个简单或复杂类型的数据，返回一个响应式的对象
  // 本质: 是将传入的数据的基础上，外层包装成一个带有value属性的对象，包成了复杂类型
  // 底层，包成复杂类型之后，再借助 reactive 实现的响应式
  // 注意点:
  // 1. 脚本中访问数据，需要通过 .value 的方式访问
  // 2. template模板中访问数据，不需要加 .value

  // 推荐: 以后声明数据，统一(优先)使用 ref
  import { ref } from 'vue'
  const count = ref(0)
  console.log(count.value)
  const setCount = () => {
    count.value++
  }
</script>

<template>
  <div>
    <!--    <div>{{ state.count }}</div>
    <button @click="setCount">+1</button> -->
    <div>{{ count }}</div>
    <button @click="setCount">+1</button>
  </div>
</template>

```

## 04.组合式API - computed

![](/img_05/06.png)

```vue
<script setup>
  // const 计算属性 = computed(()=>{
  //   return 计算返回结果
  // })

  import { computed, ref } from 'vue'
  // 声明数据
  const list = ref([1, 2, 3, 4, 5, 6, 7, 8])

  // 基于list派生一个计算属性，从list中过滤出 > 2 的数
  const computedList = computed(() => {
    return list.value.filter(item => item > 2)
  })

  // 定义一个修改数组的方法
  const addFn = () => {
    list.value.push(666)
  }
</script>

<template>
  <div>
    <div>原始数据： {{ list }}</div>
    <div>计算后的数据： {{ computedList }}</div>
    <button @click="addFn">修改</button>
  </div>
</template>
```

![](/img_05/07.png)

```js
const count = ref(1)
const plusOne = computed({
  get: () => count.value + 1,
  set: (val) => {
    count.value = val - 1
  }
})

plusOne.value = 1
console.log(count.value) // 0
```

## 05.组合式API - watch(函数)

#### ①监听单个数据

![](/img_05/08.png)

```js
   // 1. 监视单个数据的变化

```

#### ②监听多个数据

![](/img_05/09.png)

```js
  // 2. 监视多个数据的变化
  // watch([ref对象1, ref对象2], (newArr, oldArr) => {
  //   console.log(newArr, oldArr)
  // })
  watch([count, nickname], (newArr, oldArr) => {
    console.log(newArr, oldArr)
  })
```

#### ③immediate 立即执行

![](/img_05/10.png)

```js
   // 3. immediate 立即执行
   watch(count, (newValue, oldValue) => {
     console.log(newValue, oldValue)
   }, {
     immediate: true
   })
```

#### ④deep 深度监视

```js
  // 4. deep 深度监视,默认 watch 进行的是浅层监视
  // const ref1 = ref(简单类型) 可以直接监视
  // const ref2 = ref(复杂类型) 监视不到复杂类型内部数据的变化
  const userInfo = ref({
    name: 'zs',
    age: 18
  })
  const setUserInfo = () => {
    // 修改了 userInfo.value 修改了对象的地址，才能监视到(如果不用deep的话)
    // userInfo.value = { name: 'ls', age: 50 }
    userInfo.value.age++
  }
  watch(userInfo, (newVal, oldVal) => {
    console.log(newVal, oldVal)
  }, {
    deep: true
  })
```

#### ⑤监听对象的某个单个属性

![](/img_05/11.png)

```js
  const userInfo = ref({
    name: 'zs',
    age: 18
  })
  const setUserInfo = () => {
    userInfo.value.age++
  }
  // 5. 对于对象中的单个属性，进行监视
  watch(() => userInfo.value.age, (newVal, oldVal) => {
    console.log(newVal, oldVal)
  })
```

## 06.组合式API - 生命周期函数

![](/img_05/12.png)

```vue
<script setup>
  import {
    onMounted
  } from 'vue'

  // beforeCreate 和 created 的相关代码
  // 一律放在 setup 中执行

  const getList = () => {
    setTimeout(() => {
      console.log('发送请求，获取数据')
    }, 2000)
  }
  // 一进入页面就发送请求
  getList()

  // 如果有些代码需要在 mounted 生命周期中执行，则放在 onMounted 中
  onMounted(() => {
    console.log('mounted生命周期函数 - 逻辑1')
  })

  // 写成函数的调用方式，可以调用多次，并不会冲突，而是按照顺序依次执行
  onMounted(() => {
    console.log('mounted生命周期函数 - 逻辑2')
  })
</script>
```

## 15.Pinia - storeToRefs方法

#### ①为什么引入此方法？

​				(**<u>直接解构，不处理，会失去响应式</u>**)	【因为store是一个用reactive包装的对象】

​		 ![](/img_05/37.png)

```vue
<script setup>
import { storeToRefs } from 'pinia'
import { useCounterStore } from '@/store/counter'
import { useChannelStore } from '@/store/channel'
const counterStore = useCounterStore()
const channelStore = useChannelStore()

const { count, msg } = storeToRefs(counterStore)// 数据不会失去响应式
const { channelList } = storeToRefs(channelStore)// 数据不会失去响应式
const { getList } = channelStore// 方法
</script>

<template>
{{ count }}
</template>
```

## 16.Pinia - 持久化

![](/img_05/38.png)

##### ①安装

​		**npm i pinia-plugin-persistedstate**

##### ②使用

```js
import { createApp } from 'vue'

import { createPinia } from 'pinia'
import persist from 'pinia-plugin-persistedstate'
// import piniaPluginPersistedstate from 'pinia-plugin-persistedstate'
// piniaPluginPersistedstate 太长了 , 一般写为 persist

import App from './App.vue'

const pinia = createPinia()
pinia.use(persist)
// pinia.use(piniaPluginPersistedstate)

const app = createApp(App)

app.use(pinia)
app.mount('#app')
```
